## 연관관계 기본 지식
[외래키, 참조키 차이]
외래키(foreign key) : 외부에서 온 키. 부모 테이블의 참조키를 가리키는 열. 외래키는 반드시 부모 테이블의 기본키나 고유키 중 하나를 참조. 중요한 점은 외래 키가 참조하는 열이 유일성(unique)을 보장해야 함
참조키(referenced key) : 부모 테이블에서 외래 키가 참조하는 열. 일반적으로 기본키(Primary key) 또는 고유키(Unique key)로 설정

[기본키, 고유키 차이]
기본 키는 테이블에서 유일성과 NOT NULL 제약 조건을 가짐
고유 키는 유일성을 보장하지만, 기본 키와 달리 NULL 값을 허용(단, 외래 키가 참조하는 경우 NULL 값은 허용되지 않음).



1:1 
- 자식 테이블의 외래키 컬럼이 유일성 제약 조건(constraint_type = 'unique') 또는 기본키로 설정되어 있는 경우
1:n 
- 부모 테이블의 **기본 키**가 자식 테이블의 외래 키로 참조, 자식 테이블의 외래 키 컬럼이 **유일성 제약 조건(UNIQUE)**이 없는 경우
- 외래키가 
- 사용자->주문 (한 사용자가 여러 주문)
n:1 
- 1:N 관계의 반대 방향으로 해석.
- 상품 -> 카테고리 (여러 상품이 하나의 카테고리)
n:n 
- 부모테이블 참조
- 중간테이블을 통해 n:n 관계를 정의
- 



| 관계 유형   | 외래 키 참조 방향                 | 예시                       |
| ------- | -------------------------- | ------------------------ |
| **1:1** | 한 테이블이 다른 테이블의 기본 키를 참조    | `Users` ↔ `UserProfiles` |
| **1:N** | "N" 테이블이 "1" 테이블의 기본 키를 참조 | `Users` → `Orders`       |
| **N:1** | "N" 테이블이 "1" 테이블의 기본 키를 참조 | `Orders` → `Products`    |
| **N:N** | 중간 테이블이 두 테이블의 기본 키를 각각 참조 | `Students` ↔ `Courses`   |

### 연관관계별 조회 쿼리
```SQL
WITH ForeignKeys AS (
    SELECT 
        kcu.TABLE_NAME AS child_table,
        kcu.COLUMN_NAME AS child_column,
        kcu.REFERENCED_TABLE_NAME AS parent_table,
        kcu.REFERENCED_COLUMN_NAME AS parent_column
    FROM 
        information_schema.KEY_COLUMN_USAGE kcu
    WHERE 
        kcu.REFERENCED_TABLE_NAME IS NOT NULL
),
UniqueKeys AS (
    SELECT 
        tc.TABLE_NAME,
        kcu.COLUMN_NAME,
        tc.CONSTRAINT_TYPE
    FROM 
        information_schema.TABLE_CONSTRAINTS tc
    JOIN 
        information_schema.KEY_COLUMN_USAGE kcu
        ON tc.CONSTRAINT_NAME = kcu.CONSTRAINT_NAME
        AND tc.TABLE_SCHEMA = kcu.TABLE_SCHEMA
    WHERE 
        tc.CONSTRAINT_TYPE IN ('PRIMARY KEY', 'UNIQUE')
)
SELECT DISTINCT
    fk.child_table,
    fk.parent_table,
    CASE
        WHEN uq_child.CONSTRAINT_TYPE = 'UNIQUE' AND uq_parent.CONSTRAINT_TYPE = 'UNIQUE' THEN '1:1'
        WHEN uq_parent.CONSTRAINT_TYPE = 'UNIQUE' THEN '1:N'
        ELSE 'N:N'
    END AS relationship_type
FROM 
    ForeignKeys fk
LEFT JOIN 
    UniqueKeys uq_child
    ON fk.child_table = uq_child.TABLE_NAME
    AND fk.child_column = uq_child.COLUMN_NAME
LEFT JOIN 
    UniqueKeys uq_parent
    ON fk.parent_table = uq_parent.TABLE_NAME
    AND fk.parent_column = uq_parent.COLUMN_NAME;
```
